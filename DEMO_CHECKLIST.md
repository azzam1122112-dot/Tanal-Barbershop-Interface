# TANAL Demo Checklist

قائمة فحص عملية لتجربة النظام داخل الصالون قبل التشغيل الحقيقي.

## قبل التجربة

- شغّل PostgreSQL المحلي.
- تأكد من استخدام Node LTS الموجود في `.nvmrc`.
- شغّل `npm install`.
- جهّز `.env` من `.env.example` واضبط `DATABASE_URL`.
- شغّل `npm run prisma:migrate`.
- شغّل `npm run prisma:seed`.
- اختياريًا شغّل `npm run demo:seed` لإضافة بيانات تجريبية.
- شغّل `npm run dev`.
- سجّل دخول المدير من `/dashboard/login`.
- أنشئ الحلاقين الحقيقيين.
- أضف الخدمات وأسعارها.
- راجع قواعد الولاء.
- أنشئ حملة تجريبية قصيرة.

## تجربة الحلاق

- افتح `/barber/login` من الجوال.
- سجّل دخول الحلاق.
- أضف التطبيق للشاشة الرئيسية من المتصفح.
- إذا لا توجد جلسة صندوق مفتوحة، اضغط "فتح جلسة صندوق".
- ابحث برقم عميل غير موجود.
- أنشئ العميل.
- افتح صفحة العميل.
- سجل زيارة كاش.
- سجل زيارة شبكة.
- جرّب مكافأة نقاط لعميل لديه رصيد كاف.
- جرّب حملة فعالة.
- تأكد أن الخدمات غير النشطة لا تظهر للحلاق.
- أغلق جلسة الصندوق من لوحة المدير.
- حاول تسجيل زيارة بعد إغلاق الجلسة وتأكد أنها مرفوضة.
- افتح جلسة صندوق جديدة في نفس اليوم وتأكد أن التسجيل مسموح.
- تأكد أن التقرير اليومي مجرد تقرير حسب تاريخ الزيارة، وليس قفلًا لتسجيل الزيارات.
- سجّل خروج الحلاق.

## تجربة المدير

- افتح `/dashboard/login` وسجّل الدخول.
- افتح صفحة الحلاقين.
- افتح صفحة الخدمات.
- افتح صفحة العملاء.
- افتح صفحة الولاء.
- افتح صفحة الحملات.
- راجع الزيارات.
- عدّل طريقة دفع زيارة.
- عدّل مبلغ زيارة بسبب واضح.
- ألغِ زيارة بسبب واضح.
- افتح التقارير وتأكد من الكاش والشبكة والصافي.
- افتح صفحة جلسات الصندوق.
- أغلق جلسة صندوق حلاق.
- عدّل زيارة بعد إغلاق الجلسة.
- افتح تقرير تصحيحات ما بعد الإغلاق.
- افتح واتساب وجهز رسالة.
- افتح رابط واتساب يدويًا.
- علّم الرسالة كمرسلة يدويًا.
- افتح الإعدادات وراجع اسم الصالون والعملة.
- سجّل خروج المدير.

## معايير النجاح

- تسجيل زيارة عادية لا يتجاوز 45 ثانية.
- لا توجد زيارة مكررة عند الضغط مرتين.
- الكاش والشبكة يظهران بشكل صحيح.
- النقاط المكتسبة والمستبدلة صحيحة.
- الخصم لا يتكرر في الزيارة الواحدة.
- الحلاق لا يستطيع دخول الداشبورد.
- المدير لا يدخل واجهة تنال للحلاقة الرجالية كحلاق.
- واتساب لا يرسل تلقائيًا.
- لا يوجد WhatsApp API أو إرسال جماعي تلقائي.
- لا يمكن للحلاق تسجيل زيارة بدون جلسة صندوق مفتوحة.
- لا يمكن للحلاق تسجيل زيارة على جلسة صندوق مغلقة.
- يمكن فتح جلسة صندوق جديدة في نفس اليوم بعد إغلاق السابقة.
- أي تعديل بعد إغلاق جلسة الصندوق يظهر في تقرير التصحيحات.
- لا تظهر بيانات حساسة في واجهات أو APIs.
